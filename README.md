# Mistral Reddit Weekly Sentiment

This app fetches Reddit posts and comments from `r/MistralAI`, sends the dataset to an LLM for sentiment analysis, and returns JSON output containing:

- Average sentiment score (`1` to `5`)
- Weekly summary (generated by the selected LLM provider)
- Top 3 posts by upvotes (link, content, upvotes)
- Keyword mention counts and links to mentions
- Optional Google Sheets export

## Requirements

- Python 3.11+
- [uv](https://docs.astral.sh/uv/)
- Mistral AI API key for the default analysis path

## Setup

1. Copy env template:

```bash
cp .env.example .env
```

2. Fill in values in `.env`:

- `REDDIT_USER_AGENT` (recommended)
- `MISTRAL_API_KEY`

Optional provider overrides:

- `LLM_PROVIDER=mistral` or `LLM_PROVIDER=claude`
- `LLM_MODEL=` to force a specific model name
- `ANTHROPIC_API_KEY` if using Claude

Optional Google Sheets export:

- `GOOGLE_SHEETS_SPREADSHEET_ID=1WmjMDPnTO4ZyOLR0lfDQ8CB-yaRZEWj_Uv2Y58jQj8g`
- `GOOGLE_SHEETS_SUMMARY_WORKSHEET=sentiment_summary`
- `GOOGLE_SHEETS_KEYWORDS_WORKSHEET=keyword_mentions`
- `GOOGLE_SHEETS_SERVICE_ACCOUNT_FILE=/absolute/path/to/service-account.json`

Or use `GOOGLE_SHEETS_SERVICE_ACCOUNT_JSON` instead of a file path.

The spreadsheet must be shared with the Google service account email used by those credentials.

3. Install dependencies:

```bash
uv sync
```

## Run

```bash
uv run mistral-sentiment --days 7 --subreddit MistralAI --output output/weekly_sentiment.json
```

If `--output` is omitted, JSON is still printed to stdout.

Default analysis provider and model:

- Provider: `mistral`
- Model: `mistral-medium-2508` (Mistral Medium 3.1 served by Mistral AI)

To use Claude instead:

```bash
uv run mistral-sentiment --provider claude --model claude-sonnet-4-6 --subreddit MistralAI --output output/weekly_sentiment.json
```

To also write results to Google Sheets:

```bash
uv run mistral-sentiment --days 7 --subreddit MistralAI --output output/weekly_sentiment.json --write-google-sheets
```

Optional date range mode (instead of trailing `--days`):

```bash
uv run mistral-sentiment --subreddit MistralAI --start-days-ago 14 --end-days-ago 7 --output output/range_14_to_7.json
```

- `--start-days-ago` is the older boundary.
- `--end-days-ago` is the newer boundary (defaults to `0` when omitted).

Specific calendar date mode:

```bash
uv run mistral-sentiment --subreddit MistralAI --date 2026-03-10 --output output/for_2026_03_10.json
```

Specific calendar date range mode:

```bash
uv run mistral-sentiment --subreddit MistralAI --start-date 2026-03-01 --end-date 2026-03-07 --output output/for_2026_03_01_to_2026_03_07.json
```

- `--date` analyzes a single UTC calendar day.
- `--start-date` and `--end-date` are inclusive UTC dates in `YYYY-MM-DD` format.
- If `--end-date` is omitted, the app uses the same date as `--start-date`.

Google Sheets worksheets used by default:

- `sentiment_summary`
- `keyword_mentions`

If these worksheets do not already exist, the app will create them automatically.

Expected columns for `sentiment_summary`:

- `run_utc`
- `subreddit`
- `window_label`
- `window_start_utc`
- `window_end_utc`
- `posts_count`
- `comments_count`
- `analysis_provider`
- `analysis_model`
- `average_sentiment`
- `summary_of_week`
- `sentiment_method_notes`
- `top_3_posts_json`

Expected columns for `keyword_mentions`:

- `run_utc`
- `subreddit`
- `window_label`
- `keyword`
- `count`
- `links_json`

## Keywords

Edit `keywords.txt` to add or remove tracked terms. Use one term per line.

## Notes

- Reddit endpoints and rate limits can change over time.
- The app uses Reddit public JSON endpoints only (no Reddit app credentials required).
- Mistral is the default sentiment provider.
- Claude remains available as an optional provider.
- LLM input payload size is bounded by `LLM_MAX_ITEMS` and `LLM_MAX_CHARS_PER_ITEM`.
- Google Sheets export appends one row per run to `sentiment_summary` and one row per keyword per run to `keyword_mentions`.
